# Network Incident Escalation Runbook

Document ID: DT_NET_RUN_004
Company: Deutsche Telekom
Department: Network Operations
Region: Germany
Product: 5G
Access Level: support_agent

## Incident Review
Support must check active incidents before escalating individual signal cases. Relevant evidence includes postcode, serving cell, affected technology, incident start time, maintenance window, and number of similar contacts.

## Escalation Rules
Escalate to Network Operations L2 when three or more customers in the same area report degraded service, when an outage ticket already exists, or when a single enterprise customer has an SLA-impacting outage.

## Required Ticket Data
Tickets must include customer region, product, device model, SIM type, latest reprovisioning action, speed test result, and whether LTE fallback works.
